Build a cross-platform mobile expense tracking app called Kobiton Expense Tracker for iOS and Android. The app should be inspired by the workflow and screen structure of the attached Perfecto ExpenseAppVer1.0.apk screenshots, but it must use Kobiton branding, Kobiton colors, Kobiton logo/text treatment, and Kobiton naming throughout. The goal is to recreate the same general user journey and testing value while avoiding Perfecto branding.

Use React Native with Expo so the project can run in Replit and be built for both iOS and Android from one codebase. Prefer TypeScript. Use Expo Router or React Navigation, React Hook Form with Zod validation, Expo Local Authentication for biometrics, Expo Image Picker or Document Picker for receipt upload, and lightweight local persistence first. Structure the code cleanly so it can later connect to a backend API.

Product goal
Create a polished mobile app for demoing and testing expense workflows. The app must support:

Login with credentials.

Optional biometric login after first successful credential login.

Viewing a list of expenses.

Adding a new expense with form fields similar to the reference app.

Attaching a receipt image from camera or gallery for image injection and testing workflows.

Editing and deleting expenses.

Running on both iOS and Android with responsive mobile UI.

Branding
Replace all Perfecto branding with Kobiton branding.

App name: Kobiton Expense Tracker

Use Kobiton-inspired color palette: deep blue primary, teal/cyan accent, white/light gray surfaces, dark text.

Clean enterprise QA/testing product look.

Include a simple Kobiton-style wordmark in the header or splash if no official asset is provided.

Do not use Perfecto logo, name, or copyrighted visual assets.

Password field.

Checkbox or switch: “Enable Biometric”.

Login button.

Signup button, but it can open a placeholder modal saying “Signup coming soon” unless full signup is implemented.

Behavior:
